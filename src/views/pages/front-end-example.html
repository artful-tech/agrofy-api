Documentação da API

Guia de referência para os desenvolvedores do Agrofy. Estes endpoints compõem o contrato de comunicação entre o backend (Node/Prisma) e o frontend (Flutter/Dashboard).

Fazendas e Lotes

GET /api/farms

Lista todas as propriedades rurais de Maricá.

POST /api/farm

Cria uma nova propriedade agrícola no sistema.

DELETE /api/farm/:id

Exclui permanentemente uma propriedade do sistema.

PATCH /api/farm/:id

Edita os dados cadastrais de uma fazenda.

GET /api/plots

Lista lotes vinculados a fazendas específicas.

POST /api/plot

Adiciona um novo lote a uma fazenda.

DELETE /api/plot/:id

Exclui um lote vinculado a uma fazenda.

PATCH /api/plot/:id

Edita as características de um lote existente.

Catálogo e Plantio

GET /api/crops

Retorna o catálogo completo de vegetais e ciclos.

POST /api/crop

Cadastra um novo tipo de cultura no catálogo.

DELETE /api/crop/:id

Exclui uma cultura do catálogo geral.

PATCH /api/crop/:id

Edita os detalhes de uma cultura (ex: tempo de maturação).

POST /api/seasons

Inicia um novo ciclo de plantio em um lote específico.

DELETE /api/season/:id

Exclui o registro de uma safra (ciclo produtivo).

PATCH /api/seasons/:id

Atualiza o status da safra (ex: marcar como "Colhido").

Almoxarifado e Capital

GET /api/inventory

Exibe o saldo atual de sementes e insumos.

POST /api/inventory

Registra entrada de novos materiais no estoque.

DELETE /api/inventory/:id

Deleta permanentemente sementes ou insumos do estoque.

PATCH /api/inventory/:id

Edita o saldo ou detalhes de um item no estoque.

GET /api/inventory/alerts

Lista itens com estoque abaixo do nível de segurança.

Administração e Diário

POST /api/finances/expenses

Registra uma nova conta paga ou gasto operacional.

GET /api/finances/summary

Gera resumo mensal de gastos em BRL e Moeda Mumbuca.

POST /api/logs

Registra um evento ou observação climática no diário de campo.

🛡️ Nota de Segurança

As operações de DELETE e PATCH requerem verificação de integridade no banco de dados para evitar orfandade de dados vinculados a safras históricas.

💡

Lembrete para o Time:

Todos os endpoints devem utilizar tratamento de erros centralizado e as respostas devem seguir o padrão JSON. O deploy da API está sendo configurado pela Pessoa 5 (DevOps).
